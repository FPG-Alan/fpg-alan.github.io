从输入地址到获取页面详细过程

2017-03-23

之前看过一到面试题,需要尽可能的详细描述用户在浏览器中打开一个网页的过程,目前我把《HTTP权威指南》看到了第五章,在这里试着描述一下。

这个过程可以说的很简单,也可以很复杂,我认为主要还是

  1. 对HTTP乃至TCP/IP的理解
  2. 对浏览器如何解析HTML、CSS、JS等资源的理解。
    所以今天的描述只代表我目前的理解程度。
  1. 用户打开浏览器,在地址栏输入目标地址,按下回车或点击按钮。
  2. 浏览器接收目标地址,从目标地址(URL)中解析出方案(协议),端口号(默认80),以及可能会存在的查询条件。
  3. 浏览器检查缓存表中是否存在对应IP,如果存在则直接获取IP,如果不存在则连接DNS域名解析服务器,获取对应IP。
  4. 浏览器根据上一步获得的IP地址,开始尝试连接目标主机
    4.1 向目标主机发送一个40-80字节的TCP分组,带有一个SYN标志,用来说明这是一个连接请求
    4.2 如果一切顺利,目标服务器对一些连接参数进行计算,并返回一个TCP分组,这个分组中,存在SYN和ACK标志位,说明连接请求已经被接受
    4.3 浏览器向目标服务器发送一个TCP分组通知它连接已经建立
  5. TCP连接已经被建立,浏览器根据第二步中解析出的地址向目标服务器发送一个方法为 GET 的HTTP事务请求报文。
  6. 目标服务器接受请求报文, 解析请求行, 并将解析出的报文缓存在内存中, 直到收到全部请求报文并足以解析理解其意义为止。
  7. 如果一切顺利, 目标服务器解析请求报文完成, 将其中的URI映射到本地的对应地址, 并开始读取对应文件内容。
  8. 目标服务器开始构建响应报文, 包含响应状态码, 响应首部, 以及响应主体(在这个分析案例中,相应主体应该存在并且为对应站点首页)
  9. 浏览器收到响应报文, 并开始解析报文主体(这里已Chrome为例)
    9.1 呈现引擎(下文称为webkit)读取文档内容
    9.2 解析HTML文档, 转化各个节点, 构建DOM树结构
    9.3 解析内联CSS样式数据, 构建渲染树, 渲染树会包含多个带有视觉属性(如颜色和尺寸)的矩形。这些矩形的排列顺序就是它们将在屏幕上显示的顺序。
    9.4 对渲染树进行布局处理, 为每个节点分配一个应出现在屏幕上的确切坐标。
    9.5 webkit 遍历渲染树, 由浏览器的用户界面后端层将每个节点绘制出来。
    注意, 9.2-9.5是一个渐进式的过程, 为了提高用户体验, webkit并不会等待所有html内容都被解析完成后才进行绘制,而是解析一点绘制一点。
  10. 如果当前页面存在其他请求(外部CSS文件、JS文件), 浏览器会对这些文件重复步骤5-9。(这些请求一般是同步发起的)。
